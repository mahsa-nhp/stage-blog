Homepage implementatie

Reflectieverslag, week 8

08/11/2021 - 12/11/2021

Deze week beeindigd niet goed. Er is een concurrent voor VIVA MEMORIA zichtbar geworden. Daardoor ik enkele weken bezig met implementeren van Homepage was en geen werkzame hulp van mijn begeleiders ontvangen kon kon ik mijn taak niet correct afmaken en elke keer bij het goedmaken en een stuk kreeg ik een probleem met andere stuk daarom aan het einde van de week hebben mijn begeleiders besloten dat ik alleen de design ga voorstellen en mijn collega Abdurrahman is verantwoordelijk voor beide front- en backend. Zo is het misschien mogelijk dat ze hun website sneller kunnen live maken!

  • Header opnieuw geschreven
  • Header responsief voor enkele breakpoints gemaakt
  • bug van navbar met design voor hogere breedtes opgelost

Maandag:


Header opnieuw geschreven

Bij vorige versie had ik bij mobile version van searchbar een probleem. Alhoewel ik het heb aangemeld krijg ik geen hulp, na het bekijken van video's over Bootstrap 5 ben ik besloten om het opnieuw schrijven. Uiteindelijk is mijn probleem opgelost geworden.

Mijn grotere probleem is het werken met localhost. Ik weet niet waarom het zo traag gaat worden met React! Elke keer bij een kleine verandering moet ik een nieuw raam openmaken en op f12 drukken om web developers tools open te kunnen maken daarom het prcces héél vervelend en tijdsverpillend is. Probleem aan mijn begelijder voorgesteld maar geen hulp!



Dinsdag:


Header responsief voor enkele breakpoints gemaakt

Ik heb eerst geprobeerd hele pagina voor deze breakpoints te schrijven

                /* portrait phones */
                @media (max-width: 575px) {
                    
                }

                /* landscape phones */
                @media (min-width: 576px) and (max-width: 767px){
                  
                }

                /* tablets */
                @media (min-width: 768px) and (max-width: 991px){
                  
                }

                /* large desktops */
                @media (min-width: 1200px) {
                  
                }

                /* xlarge desktops */
                @media (min-width: 1400px) {
                  
                }

Aangezien responsief maken altijd mislukte voor verschillende devices, ben ik besloten om het niet met localhost maar bij aantal breakpoints die bij mijn device gemaakt kunnen worden maken(ik kan mijn scherm voor aantal verschillende breakpoints aanpassen). Daarom heb ik het zo gemaakt en deployed maar daarna dacht ik dat het met officiele breakpoints moet geschreven worden!

Door een conflict (die ik niet kon oplossen) heb ik mijn css file local verloren daarom misschien nog aantal andere breakpoints zijn er die ik vergeten ben!

Woensdag:


Bug van navbar met design voor hogere breedtes opgelost

Design van navbar had een probleem voor hogere breakpoints. Ik was de hele dag ermee bezig. Uiteindelijk heb ik het opgelost en ook voor alle breakpoints responsief gemaakt maar tijdens het deployen krijg ik een onlogische conflict (conflict met mijn eigen codes die voor navbar geschreven waren maar ik die helemaal verwijderd had om het opniuew te schrijven) en na een uur zoeken en verschillede pogingen het lukte niet (conflict was niet duidelijk om opgelost te worden ik had een oplossing van bitbucket gevolgd maar de conflict komt weer.). Ik kon mijn begeleider ook niet bereiken om het op te lossen. Ten slote met een foute beslissing heb ik mijn codes local verloren.

Thuis heb ik geprobeerd om een reverse bij mijn vorige branch te maken maar krijg ik weer de conflict daarom ben ik besloten om het volgende keer aan mijn begeleiders tonen, mss kunnen ze het oplossen maar was ook een foute beslissing!


Donderdag:


Vandaag was een vrije dag maar ik ging van 13:00 tot 17:00 op kantoor om met mijn begeleiders over de website en wat kan beter of toegevoed worden bespreken!

Ik heb mijn begeleiders gevraagd om mijn probleem over conflict op te lossen maar volgens hen zulke problemen kon ik met mijn collega Abdurrahman bespreken. Ik heb voorgesteld dat ik deze repo op mijn device verwijderen en een nieuwe clone te maken en verder werken (met reverse op vorife stabiele versie) en ze gingen akkoord ermee.

We hebben ook besproken over mijn werk dat het zolang duurde en kreeg een deadline tot maandag dat navbar klaar moet zijn. Ik heb beloofd dat hele homepage tegen maandag klaar kan zijn. Daarna hebben we over de website gesproken en alles was OK gelopen.

Nadat ik thuis ben gekomen heb ik op onze groep in WhatsApp een bericht ontvangen dat ik aan design moet verder werken en momenteel niet meer op front-end. Omdat er geen tijd is dat ik rustig met testen(goed of fout maken) de design implementeren. Vanaf nu is Abdurrahman ervoor verantwoordelijk!


Vrijdag:


Vrijdag is een vrije dag. Aangezien ik van mijn taak(front-end) opzij gelaten ben geworden was ik vrij.
