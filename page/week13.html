Reflectieverslag, week 12

13/12/2021 - 17/12/2021

Deze week ging het vrij goed. Voor aantal problemen en taken heb ik veel onderzoek gedaan. Bij sommige was het heel complex en alhoewel ik veel documentaties heb gelezen kon geen manier vinden om mijn probleem op te lossen. Aan de andere kant bij onderzoek over payment service providers heb ik veel over dit systeem en services geleerd waar ik vroeger geen idee erover had.

Ik ben ook weer begonnen met implementatie van pagina's wat na anderhalf maand afstand heel leuk was en ik kon het heel simpler dan vroeger implementeren.

Volgende taken heb ik gedaan:

  • geprobeerd een manier te vinden om ongeldige e-mailadressen bij registraties niet accepteren
  • Nederlandse versie van errors en reset password
  • taak gekregen om met linkgenerator aantal links te maken
  • regex codes gevonden voor form validaties
  • basic form voor "herdenkingsruimte aanmaken" gemaakt
  • layout van verification email veranderd
  • onderzoek over payment service providers
  • test van functionaliteit
  • implementatie van privacybeleid pagian

Maandag:

  • geprobeerd een manier te vinden om ongeldige e-mailadressen bij registraties niet accepteren

    Ik kon alleen javascript gebruiken bij custom UI van Auth0, maar kon geen manier ervoor vinden.

    Er is een betalen service dat bij server moet staan. Het gaat automatisch ongeldige e-mails negeren. Er is ook een andere niet betalend manier maar moet met node.js geschreven worden, dus bij custom UI is niet mogelijk om geschreven te worden.

  • Nederlandse versie van errors en reset password

    Ik had ook een andere taak om de reset password en error messages in het Nederlands te schrijven. ik heb de reset password zinnen in het Nederlands geschreven maar bij het testen kreeg ik geen email. Deze functie werkte vorige weerk daarom dacht ik dat de codes zijn misschien aangepast. Voor de zekerheid heb ik het met mijn eigen App getest maar daar kreeg ik ook geen verification email. Daarna dacht ik dat alles door de custom UI versie komt en heb de universal login form geprobeerd maar daarmee kreeg ik weer hetzelfde probleem. Ik dacht dat misschien het probleem tijdelijk is, dus heb tot volgende dag gewacht en dat klopte.

Dinsdag:

  • taak gekregen om met linkgenerator aantal links te maken

    Door aanvraag van mijn mentor heb ik via linkgenerator links gemaakt voor twee plaatsen in herdenkingsruimte. Ik heb ook via een normale <a> tag hetzelfde functie gemaakt om te tonen dat het met normale html-codes ook mogelijk is.

Woensdag:

  • reset passwoord in het Nederlands gemaakt

    De form en zijn error berichten staan nu in het Nederlands.

  • layout van verification email veranderd

    Ik heb in het Nederlands vertaald, onze logo en kleur ook gebruikt. Het is momenteel niet bruikbaar omdat ik ons email provider bij Auth0 moet toevoegen en aantal aanpassingen erbij maken.

  • poging tot Nederlandse error messages bij login pagina te hebben

    Ik heb meerdere keren geprobeerd om error messages van login page in het Nederlands te vertalen maar geen success ermee. Op gewone html en op localhost werkt de gewoone Bootstrap maar kon helaas niet functioneren bij Auth0. Ik moet later dit probleem oplossen.
